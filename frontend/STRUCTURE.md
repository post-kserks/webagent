# Предложенная структура сайта

## Информационная архитектура

1. Дашборд
- Статус агента
- Интервал опроса
- Количество задач в очереди

2. Задачи
- Конструктор `TASK/FILE/TIMEOUT/CONF`
- Для `TASK`: загрузка списка `.mp4` из GitHub-репозитория и выбор конкретного файла
- Для `TASK`: синхронизация выбора в локальный API агента
- Валидация параметров
- Отправка задачи и контроль статуса

3. Журнал
- История действий оператора
- История отправок и ошибок

## Что добавить дальше (по текущим файлам проекта)

1. Просмотр `agent.log` в UI
- Файл уже используется в агенте как основной источник диагностики.

2. Модуль управления `TIMEOUT`
- Отдельный быстрый контрол для изменения `request_interval`.

3. Карточка сессии
- Отображение `access_code`, статуса регистрации, времени последнего poll.

4. Видео-панель
- Предпросмотр последнего `screamer.mp4` (или ссылки на скачанный файл),
  так как `TASK` в текущей реализации завязан на видео.

5. Конфигуратор `config/config.json`
- Форма редактирования `uid`, `server_uri`, `request_interval`, `max_retry_interval` с экспортом JSON.

6. История задач по `session_id`
- Отдельная таблица для трассировки полного цикла задачи.
