# Frontend MVP для WEB-AGENT

Минимальный интерфейс управления агентом с упором на выбор файлов и формирование задач.

## Что уже есть

- Блок профиля агента (`UID`, `descr`, API URL, polling interval)
- Конструктор задач (`TASK`, `FILE`, `TIMEOUT`, `CONF`)
- Для `TASK`: загрузка списка видео из GitHub-репозитория и выбор конкретного `.mp4`
- Очередь задач и журнал действий
- Переключение `Mock/Live` для безопасной отладки без реального сервера
- Синхронизация выбранного видео с локальным API агента (адрес задаётся в поле `Local Control API`)

## Запуск

Откройте `frontend/index.html` в браузере.

## Что подключать к реальному API

Сейчас отправка задач в `Live` режиме уходит на:

- `POST {baseApi}wa_task/`

Рекомендуем добавить на сервере отдельную ручку для постановки задач оператором, например:

- `POST {baseApi}wa_task_push/`

чтобы не смешивать агентский poll-эндпоинт и операторский интерфейс.

## Важное замечание по текущему агенту

В текущем C++ коде `FILE`-задача отправляет только `agent.log` и не читает список файлов из `options`.
Для полной поддержки выбора файлов из UI потребуется доработка агента.

Для `TASK` агент хранит последний выбранный файл в `.last_selected_video`.
Если внешний сервер присылает `options = null`/`""`, агент использует это сохранённое значение.
